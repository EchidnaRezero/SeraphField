# Seraph Field 모바일 사양

모바일에서 사이트가 어떻게 보이고 어떻게 움직이는지 따로 정리한 문서입니다.

공통 구조와 데스크톱 중심 설명은 `docs/SITE_SPEC.md`를 기준으로 봅니다.

## 1. 목표

- 모바일에서 읽기와 조작이 쉬운 화면을 만든다.
- 데스크톱 UI는 그대로 두고, 모바일에서만 필요한 구조와 효과를 따로 조정한다.
- 공통 로직은 재사용하고, 모바일에서는 레이아웃과 표시 방식만 바꾼다.

## 2. 모바일 공통 원칙

- 모바일에서는 무거운 화면 효과를 끈다.
  - `CustomCursor`
  - `QuantumField`
  - 화면 전환용 `AnnihilationTransition`
- 모바일에서는 긴 콘텐츠를 `박스 안 드래그`보다 `페이지 스크롤`로 읽게 한다.
- 버튼과 토글은 손가락으로 누르기 쉬운 크기를 우선한다.

## 3. 로비

- 모바일 로비는 상단 정보 카드 뒤에 여백을 두고, 그 아래에 카테고리 2x2 그리드를 둔다.
- 검색 박스는 카테고리 그리드 바로 아래에 둔다.
- 모바일 로비에서는 `Tracked Repositories` 패널을 보여주지 않는다.
- 데스크톱에만 있는 우측 패널 구조를 모바일에 그대로 옮기지 않는다.

## 4. 아카이브

- 모바일 아카이브는 문서 목록 중심이 아니라 본문 중심 구조를 쓴다.
- 모바일에서는 데스크톱의 좌측 문서 리스트를 따로 보여주지 않는다.
- 상단 검색은 현재 아카이브 내부 필터가 아니라 검색 결과 화면으로 이동하는 검색 진입점으로 동작한다.
- `Document Map`은 접힌 상태로 시작하고, 필요할 때만 펼친다.
- `Series and Groups`도 접힌 상태로 시작한다.
- `Series and Groups`를 연 뒤에도 `Series`, 각 `Group`은 각각 따로 접고 펼친다.

## 5. 검색 결과

- 모바일 검색 결과는 결과 목록 자체가 세로 스크롤되도록 구성한다.
- 결과 카드는 바깥 카드 클릭과 태그 버튼 클릭이 충돌하지 않도록 중첩 버튼 구조를 쓰지 않는다.
- 태그, 시리즈, 그룹 버튼은 카드 안에서 따로 눌러 검색을 다시 열 수 있다.

## 6. Mermaid

- 모바일에서 `flowchart LR`, `flowchart RL`, `graph LR`, `graph RL`은 세로 읽기가 쉽도록 `TB` 흐름으로 바꿔 렌더링한다.
- 세로형 Mermaid는 작은 팬 뷰어 안에 가두지 않고 본문처럼 아래로 이어서 보여준다.
- 모바일 세로형 Mermaid는 `-`, `+`, `reset` 버튼으로만 확대와 축소를 조절한다.
- 모바일에서는 드래그 팬보다 페이지 스크롤을 우선한다.

## 7. 수식

- 모바일에서 긴 display math는 폭에 맞게 자동 축소한다.
- display math가 일반 문단에 있든, 인용문 안에 있든 같은 기준으로 보정한다.
- 자동 축소 뒤에도 너무 긴 경우에는 가로 스크롤이 마지막 안전장치가 된다.
- 인용문 안 예시 문장은 본문보다 과하게 튀지 않게 본문 크기에 가깝게 맞춘다.

## 8. 확인 항목

모바일 수정 뒤에는 아래를 같이 본다.

1. 글 제목이 과하게 크지 않은가
2. 긴 수식이 오른쪽으로 튀어나오지 않는가
3. Mermaid 아래쪽 항목을 페이지 스크롤로 자연스럽게 볼 수 있는가
4. 검색 결과 목록이 끝까지 스크롤되는가
5. 아카이브에서 검색 입력이 검색 결과 화면으로 이어지는가
6. 접힘 패널이 한 번에 전부 열리지 않고 의도한 단위로만 열리는가
7. 같은 화면을 데스크톱 폭으로 다시 봤을 때 기존 UI가 유지되는가

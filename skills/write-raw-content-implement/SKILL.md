# Write IMPLEMENT Content

Write `RAW/IMPLEMENT/**/*.md` as hands-on execution records.
Keep `SKILL_KR.md` synchronized whenever the substantive instructions in this file change.

## Focus

- Record only work that was actually performed.
- Include environment, inputs, settings, outputs, and interpretation when they matter.
- Use this category when the code is yours or the practice work is yours.
- Link to `REPO` only when you need to point back to external code you analyzed.
- Link to `THEORY` for generalized explanations instead of reteaching theory inside implementation notes.

## Preferred Structure

- goal
- environment and inputs
- commands or settings actually used
- observed outputs or metrics
- interpretation
- next concrete question if needed

## Evidence Rules

- Ground claims in actual runs, logs, saved outputs, metrics, or artifacts.
- Distinguish observation from interpretation.
- Do not invent results that were not actually produced.

# Scratch To RAW Pipeline

`SKILL.md`의 실질적인 지침이 바뀌면 이 문서도 함께 갱신합니다.

## 목표

이 스킬은 거친 초안이나 원본 메모를 사이트에 올릴 수 있는 최종 `RAW/**/*.md` 문서로 정리할 때 씁니다.

이 스킬은 직접 세부 규칙을 많이 들고 있는 스킬이 아니라, 기존 RAW 작성·점검 스킬을 순서대로 엮는 오케스트레이션 스킬입니다.

## 저장 단계

- 저장 위치는 세 단계로 나눕니다.
  - `SCRATCH/**/*.md`: Git으로 추적하지 않는 거친 private 초안
  - `DRAFT/**/*.md`: Git으로 추적하고 공개 저장소에 push해도 되지만, 아직 사이트에는 게시하지 않는 작업중 버전
  - `RAW/**/*.md`: 사이트 게시용 최종 원본
- `docs/`는 기본 초안 보관 위치로 쓰지 않습니다.
- `docs/`는 사람용 프로젝트 문서용으로 두고, 거기에 남는 파일은 문서이거나 의도된 아카이브 자료일 때만 둡니다.
- 예전 초안이 아직 `docs/` 아래에 남아 있으면, 일반적인 초안→RAW 흐름을 돌리기 전에 `SCRATCH/` 또는 `DRAFT/`로 옮기거나 그 기준으로 다시 씁니다.
- 이미 게시했던 `RAW` 문서를 나중에 다시 손보는 경우, 먼저 `DRAFT/`로 내려서 작업한 뒤 다시 재게시해도 됩니다.

## 파이프라인

1. 초안을 읽고 어떤 종류의 원본인지 먼저 파악합니다.
   - 거친 Markdown 초안
   - 여러 역할이 섞인 공부 메모
   - 이론 노트
   - 논문 리뷰 초안
   - 외부 코드 독해 초안
   - 구현/실험 메모
2. 현재 원본이 `SCRATCH/`, `DRAFT/`, `RAW/` 중 어디에 있어야 하는 상태인지 먼저 판단합니다.
3. 어떤 파일이 `DRAFT/`로 올라가기 전에는, 공개 안전, frontmatter, 링크, 파서 호환 저장소 규칙에 대해 먼저 `AGENTS.md`를 적용합니다.
4. 이 초안을 한 문서로 둘지, 여러 최종 문서로 나눌지 결정합니다.
5. 최종 문서마다 RAW 카테고리를 정합니다.
   - 내용이 분명한 수학과 스타일의 이론 노트면 `write-raw-content-theory`를 적용합니다.
   - `PAPER`, `REPO`, `IMPLEMENT`는 아직 여기서 같은 수준으로 고정된 기준을 들고 있지 않으니, 사용자의 지시를 따릅니다.
6. 최종 문서 묶음 전체에는 `write-raw-content-common`을 적용합니다.
7. 문서에 수식 표기가 있으면 `write-math-notation`도 같이 적용합니다.
8. 마무리 전에 `check-site-content`를 적용합니다.

## 분리 규칙

- 여러 역할이 섞인 초안을 억지로 한 개의 최종 RAW 문서로 몰아넣지 않습니다.
- 초안 안에 여러 역할이 있으면, 주된 역할 기준으로 여러 문서로 나누고 서로 링크합니다.
- 다만 한 역할이 아주 짧은 보조 설명 수준이면 무조건 분리하지 말고, 그 보조 부분이 문서의 중심을 흔들기 시작할 때 분리합니다.

## 카테고리 규칙

- `THEORY`
  - 현재 확정된 프로젝트 기준인 `write-raw-content-theory`를 사용합니다.
- `PAPER`
  - 현재는 사용자의 지시를 따릅니다.
- `REPO`
  - 현재는 사용자의 지시를 따릅니다.
- `IMPLEMENT`
  - 현재는 사용자의 지시를 따릅니다.

## 출력 규칙

- 거친 private 재료는 추적 가능한 초안이 되기 전까지 `SCRATCH/`에 둡니다.
- 개인식별 정보, 로컬 경로, private 메모 같은 공개 위험 요소를 지운 뒤에만 `DRAFT/`로 올립니다.
- 추적하지만 아직 사이트에는 게시하지 않는 작업중 문서는 `DRAFT/`에 둡니다.
- 최종 문서는 맞는 `RAW/` 카테고리 경로 아래에 저장합니다.
- 변경 설명이나 전후 비교가 아니라 최종 내용만 씁니다.
- 결과물은 저장소에 그대로 공개해도 되는 형태여야 합니다.

## 저장 전 최종 점검

- `AGENTS.md`를 적용했는지 확인합니다.
- 섞인 초안을 필요할 만큼 분리했는지 확인합니다.
- 최종 카테고리 결정이 분명한지 확인합니다.
- `write-raw-content-common`을 적용했는지 확인합니다.
- 분명한 이론 문서라면 `write-raw-content-theory`를 적용했는지 확인합니다.
- 문서에 수식이 있다면 `write-math-notation`을 적용했는지 확인합니다.
- 마무리 전에 `check-site-content`를 적용했는지 확인합니다.

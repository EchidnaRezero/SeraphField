# Check Project Docs

`SKILL.md`의 실질적인 지침이 바뀌면 이 문서도 함께 갱신합니다.

## 목표

프로젝트 문서를 저장, 커밋, 게시하기 전에 중요한 문제만 골라 검수합니다.

우선적으로 보는 것:

- 중복되거나 겹치는 내용
- 애초에 안 적어도 되는 말, 예를 들어 이미 지워졌거나 없어져 버린 내용을 굳이 언급하는 문장, 문서만 보면 바로 아는 말을 반복하는 문장
- `이 문서는 ...`처럼 문서 자신에 대해 설명하는 문장
- `AGENTS.md`에서 추적 문서나 공개 문서에 쓰지 말라고 한 정보
- 문서 사이의 불일치
- 영어본과 한국어본의 의미 어긋남
- 문서 책임 경계가 흐린 부분

## 기본 검수 범위

다음 같은 문서에 이 스킬을 씁니다.

- `README.md`
- `docs/**/*.md`
- `AGENTS.md`
- `AGENTS_KR.md`
- `skills/**/SKILL.md`
- `skills/**/SKILL_KR.md`

Git으로 추적되는 공개용 Markdown은 공개 저장소 문서로 취급합니다.

## 검수 방식

- 결과는 발견 사항부터 씁니다.
- 심각도 순서로 정리합니다.
- 파일 경로를 정확히 적습니다.
- 개요는 짧게 유지합니다.
- 단순 트집잡기나 특정 문구의 억지 해석이 아니라, 실제 동작, 렌더링, 수식 출력, 파싱, 사용자 이해에 진짜 영향을 주는 문제인지 먼저 판단하고, 그 영향이 크지 않다면 보고하지 않습니다.

## 서브에이전트 분담

항상 파일별이 아니라 위 체크리스트 그룹별로 나눠 서브에이전트 검수를 합니다.
각 서브에이전트에는 대상 파일과 맡은 그룹만 주고, 파일 경로가 붙은 발견 사항만 받아서 마지막에 겹치는 항목을 합칩니다.
현재 환경에서 서브에이전트를 쓸 수 없으면 단일 에이전트 검수로 조용히 넘기지 말고, 검수가 막혔다고 바로 보고합니다.

## 그룹별 체크리스트

### 그룹 1. 구조와 책임 범위

확인할 것:

- 이 문서에 있어야 할 내용과 다른 문서로 가야 할 내용이 잘 나뉘는가
- 한 문서가 두 가지 일을 동시에 하려고 하지 않는가
- 표, 플로우차트, heading 구조가 이미 보여 주는 내용을 prose가 반복하지 않는가
- `이 문서는 ...`, `이 절은 ...`처럼 문서나 절 자체를 설명하는 문장이 들어가 있지 않은가
- 섹션 순서가 실제 읽는 흐름과 맞는가

### 그룹 2. 중복과 드리프트

확인할 것:

- 한 문서 안에서 같은 규칙이나 설명이 두 번 나오지 않는가
- 두 문서가 표현만 조금 바꿔 같은 내용을 거의 반복하지 않는가
- 한 파일이 다른 파일을 요약하면서 오래된 표현이나 잘못된 설명을 남기지 않았는가
- 맵, 체크리스트, 요약문이 현재 원본 파일 상태와 어긋나지 않는가

### 그룹 3. `AGENTS.md` 정보 규칙

확인할 것:

- `AGENTS.md`가 추적 문서나 공개 문서에 쓰지 말라고 한 정보가 들어가 있지 않은가
- 공개용 파일에 개인 식별 정보, 로컬 전용 경로, private 메모, 프롬프트 조각, AI 작성 흔적이 없는가
- 추적되는 문서에 프로젝트 규칙상 바꿔야 하는 계정 비슷한 표기가 그대로 남아 있지 않은가
- 외부 로컬 경로나 private 계정 정보가 승인 없이 문서에 적히지 않았는가

### 그룹 4. 형식과 렌더링

확인할 것:

- Markdown 구조가 문서 용도와 맞는가
- 표, heading, 코드 블록, Mermaid 블록 문법이 일관적인가
- `RAW/**/*.md`가 사이트 파서와 렌더러 제약을 계속 지키는가
- 문법상 맞더라도 읽기 어려운 형식 선택이 남아 있지 않은가

### 그룹 5. 파일 간 일관성

확인할 것:

- 영어본과 한국어본이 같은 뜻을 유지하는가
- 파일명, 스킬명, 경로 링크가 현재 프로젝트 경로를 제대로 가리키는가
- 카테고리명, 워크플로명, 용어가 문서마다 다르게 흔들리지 않는가
- 어떤 문서가 말한 책임 경계를 다른 문서가 깨고 있지 않은가

## 예외사항

- `PAPER`, `REPO`, `IMPLEMENT`에 스킬 파일이 이미 있어도, 다른 문서에서 그 카테고리 기준이 아직 `THEORY`처럼 확정되지 않았다고 적혀 있으면 그 점만으로 `불일치`나 `충돌`로 보고하지 않습니다. 현재는 그쪽 실제 문서가 아직 적어서 스킬 파일이 먼저 있는 상태이기 때문입니다.
- `AGENTS.md`에 영어 문서가 AI용 원본이고 `_KR`는 같은 의미의 사람용 짝 문서라고 적혀 있어도, `AGENTS_KR.md`가 그 점을 사람용 문서라서 일부러 다시 적지 않은 경우는 드리프트로 보고하지 않습니다.

## 최종 보고 규칙

최종 보고는 아래 순서로 씁니다.

1. 발견 사항
2. 열린 질문이나 가정

각 발견 사항에는 아래를 넣습니다.

- 충돌, 중복, 드리프트, 책임 범위 문제 중 무엇인지
- 먼저 쉬운 말로 사용자 영향
- 그 다음 기술적인 이유
- 정확한 파일 경로
- 애매할 수 있는 경우에는 왜 예외사항에 해당하지 않는지도 짧게 적습니다.

# Write REPO Content

`SKILL.md`의 실질적인 지침이 바뀌면 이 문서도 함께 갱신합니다.

## 중심

- 실제 파일, 함수, 클래스, 노드, 스크립트, 옵션에서 시작합니다.
- 코드에서 어떤 결정이 어디서 이뤄지고 경로가 어떻게 흐르는지 설명합니다.
- 수학이 중심이 되면 `THEORY` 문서로 연결합니다.
- 이 카테고리는 외부 코드 분석용이지, 내 구현 작업 기록용이 아닙니다.

## 권장 구조

- 저장소/버전 범위
- 진입 경로 또는 파일 배치
- 중요한 분기 지점
- 핵심 함수, 클래스, 노드
- 그 코드가 시사하는 동작

## 근거 규칙

- `test/` 아래 실제 체크아웃 코드를 근거로 적습니다.
- 필요할 때 파일, 함수, 클래스, 노드, 스크립트, 플래그를 구체적으로 적습니다.
- `tracked_versions`는 저장소 버전 출력에 참여해야 할 문서에서만 사용합니다.
